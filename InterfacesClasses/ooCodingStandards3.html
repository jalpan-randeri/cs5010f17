Coding Standards Illustrated 

      // A Foo is an object of any class that implements Foo.
      // Module such-and-so expects to work with a list of Foo.
      
      interface Foo {
      
          // purpose statement goes here...
      
          int m1;
      
          // purpose statement goes here...
      
          Foo addBar (Bar b);
      }

Data definitions go with interfaces. 

      // Constructor template for Class1:
      //     new Class1 (a, b, c)
      // Interpretation:
      //     an object of Class1 represents a ....

Classes have constructor templates and interpretations. 

      class Class1 implements Foo {
          int a;        // interpretation goes here...
          boolean b;    // interpretation goes here...
          Foo c;        // interpretation goes here...
      
          // interpretation of LOCAL_CONSTANT goes here...
          static final int LOCAL_CONSTANT = ...;

Constants are declared static final. Constants need interpretations too. 

          // purpose statement goes here...
          public int m1 ();
      
          // purpose statement goes here...
          public Foo addBar (Bar b) { ... }

No public methods except those listed in the interface. Public methods come before non-public methods. 

          // purpose statement goes here...
          private int method1 (...) { ... }
      
          // purpose statement goes here...
          private int method2 (...) { ... }
      }

Private methods need purpose statements just like public methods.
