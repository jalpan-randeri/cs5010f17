The Method is Determined by the Class of the Receiver 

          Interface1 obj1 = new Class1 (15, 35, 5);
          Interface1 obj2 = new Class2 (15, 35, 5);
          Interface1 obj3 = new Class3 (15, 35, 5);

In a method call such as 

          obj1.bar(3)

we say the value of obj1 is the receiver of the message bar. We say the call passes 3 as an argument to the method determined by the receiver and the message.

There are three bar methods defined in Class1, Class2, and Class3. Which method is actually called?

The method that is actually called is determined by the class of the receiver.

In obj1.bar(3), the receiver is an object of Class1, so the bar method of Class1 is called, causing this to be bound to the receiver and the formal parameter n to be bound to 3: 

          public int bar (int n) {
              return r + n;
          }

The r field of this is 5, so obj1.bar(3) evaluates to 8.
