Variables Are Not Objects

Variables are expressions, not objects. An object can be the value of a variable, but an object cannot be a variable.

In languages such as Java, every expression has a type, but objects do not have a type. This fact will become more important when we study inheritance.

For now, be sure you understand the following example: 

          Interface1 obj1 = new Class1 (15, 35, 5);
          Interface1 obj2 = new Class2 (15, 35, 5);
          Interface1 obj3 = new Class3 (15, 35, 5);
          Interface1 obj4 = obj3;

Those four variable declarations declare four different variables, but they create only three objects. All four variables have the same type. The values of the first three variables are distinct, and are objects of three different classes. The fourth variable, obj4, has the same value as obj3.
