Step 5: Program Review

Program review is the same as before, with one new step:

The Program Review Recipe
1. Do all tests pass?
2. Are contracts accurate? Do they refer to interfaces rather than classes?
3. Are purpose statements and interpretations clear and accurate?
4. Are there ugly pieces of code that should be broken out into separate methods?
5. Are there pieces of code that are duplicated (or almost duplicated) and should be made into independent methods?
6. Does your design follow the principles of good OO design? (See previous slide.)
