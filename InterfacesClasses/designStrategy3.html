Properties of Good OO Design

This is not a course on OO design, but we can state some general principles. If you stray too far from these, that is a sign of bad design.

  • One bundle of operations = one interface
    • If the interface consists of two kinds of things, working on disjoint pieces of data, consider splitting it.
  • One structure = one class
  • Keep interfaces as small as possible.
  • Keep operations near their data.
  • Keep values local where possible.
  • Our older design principles still hold.
    • contracts, purpose statements, and invariants
    • If not every combination of values is meaningful, write an invariant (precondition) to document this.
