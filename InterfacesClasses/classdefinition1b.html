// You're looking at a file named Class1.java,
      // which defines a class named Class1 that implements
      // the interface named Interface1.
      //
      // Ordinarily, we would write a purpose statement
      // for each field and for each method, but we're leaving
      // them out to emphasize the simplicity of Java class
      // definitions.
      //
      // Java class definitions aren't as simple as they could be,
      // because you (usually) have to write a Java constructor.
      // The syntax of a constructor definition resembles
      // a method definition, but there are two differences:
      //
      //     a constructor has no return type
      //     the constructor's name is the name of the class
      //
      // Constructors take arguments specifying the values of the
      // class's fields, and the constructor body ordinarily contains
      // one line initializing each field, as shown below.
      
      class Class1 implements Interface1 {
      
          int x;
          int y;
          int r;
      
          // This is the constructor definition.
      
          Class1 (int x, int y, int r) {
              this.x = x;
              this.y = y;
              this.r = r;
          }
      
          // These are the method definitions.
      
          public int foo () {
              return x + y;
          }
      
          public int bar (int n) {
              return r + n;
          }
      
          public int baz (int n) {
              return this.bar(n) + n;
          }
      }
