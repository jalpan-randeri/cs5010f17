Quicksort: inductive proof of correctness 

      
      
      ;;; things is a list of length n+1
      ;;; list? is a total order on those things
      
      
      (define (quicksort things less?)
        (if (< (length things) 2)
            things
            (let ((pivot (first things))
                  (other (rest things)))
              (let-values (((below above)
                            (partition (lambda (x)
                                         (less? x pivot))
                                       other)))
                (append (quicksort below less?)
                        (list pivot)
                        (quicksort above less?))))))

For lists of length 2 or greater, the highlighted code will be executed.

That code gives the name pivot to the first thing in the list, and the name other to the other things in the list.

The let-values syntax gives the names below and above to the two values returned by the call to partition, which is a higher-order function that's just like filter except it returns two lists instead of just one. The first of those lists is the list that would have been returned by a call to filter. The second of those lists is the list of elements that were in the second argument but didn't make it into the first list (because they weren't less than the pivot).

So below is the list of other elements that are less than the pivot according to the total order less?, and above is the list of other elements that aren't less than the pivot.
