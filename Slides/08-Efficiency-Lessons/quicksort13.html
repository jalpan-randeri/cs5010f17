Quicksort: run time for the worst case

For a sorted list, the length of the list passed to quicksort decreases by 1 on each recursive call. That means there will be n calls that pass a non-empty list of things to sort.

Each of those calls takes time roughly proportional to the length of the list, not including the recursive calls that pass a non-empty list.

That means the total execution time is roughly proportional to the sum of the following numbers: 

      n
      n-1
      n-2
      ...
      3
      2
      1
      ———————————
      n (n+1) / 2

So quicksort takes Θ(n2) time when sorting an already-sorted list.

That is the worst case for quicksort.
