On a problem of this size, we would ordinarily expect an algorithm that runs in Θ(lg n) time to run a lot faster than just 25 times as fast as a Θ(n) algorithm.

The reason our divide-and-conquer algorithm ran only 25 times as fast is that multiplication of large numbers cannot be done in O(1) time. In both algorithms, the running time is dominated by multiplications that involve large numbers, so reducing the number of multiplications that involve small numbers doesn't help as much as you'd expect.
