Quicksort: run time for the worst case 

      
      
      ;;; things is a sorted list of length n
      ;;; list? is a total order on those things
      
      
      (define (quicksort things less?)
        (if (< (length things) 2)
            things
            (let ((pivot (first things))
                  (other (rest things)))
              (let-values (((below above)
                            (partition (lambda (x)
                                         (less? x pivot))
                                       other)))
                (append (quicksort below less?)
                        (list pivot)
                        (quicksort above less?))))))

For a sorted list of things, the pivot will be the least element in the list, so the list of other elements that lie below it will be empty. Sorting that empty list will be done in O(1) time.

The list of elements that lie above the pivot will have length n-1. That means the highlighted call to quicksort is sorting a list of length n-1.
